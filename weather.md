Tetris Weather

用「俄罗斯方块」的视觉与规则，表达天气状态的一款游戏化天气工具 App + Widget

⸻

一、产品定位

1.1 产品一句话定义

Tetris Weather 是一款用俄罗斯方块动态堆叠来“感知天气”的情绪化天气工具。

不是看数据，而是看“天气状态”。

⸻

1.2 目标用户
	•	🇷🇺 俄罗斯及高纬度地区用户
	•	每天都会查看天气
	•	喜欢 Widget / 桌面组件
	•	不追求专业气象，但在意体感和趋势

⸻

1.3 使用场景
	•	出门前 3 秒扫一眼今天天气好不好
	•	桌面 Widget 随时感知天气变化
	•	情绪反馈：今天是不是“乱堆的一天”

⸻

二、核心设计理念

2.1 设计关键词
	•	游戏化（Gamified）
	•	情绪化（Emotional）
	•	极简（Minimal）
	•	符号化（Symbolic）

⸻

2.2 为什么是俄罗斯方块

俄罗斯方块特性	对应天气意义
下落	时间流逝
堆叠	天气累积状态
消行	好天气 / 稳定期
混乱	极端天气

用户不是“读数值”，而是“看局面”。

⸻

三、天气数据 → 方块映射规则（核心）

3.1 温度（Temperature） → 方块颜色

温度区间	方块颜色
≤ -20°C	深蓝 / 紫蓝
-20 ~ 0	蓝
0 ~ 10	青绿
10 ~ 25	绿 / 黄
≥ 25	橙 / 红

体感温度优先于实际温度

⸻

3.2 降水（Rain / Snow） → 下落速度 + 密度
	•	小雨 / 小雪：
	•	方块下落慢
	•	间隔大
	•	暴雨 / 暴雪：
	•	方块高速下落
	•	同时出现多个方块

⸻

3.3 风力（Wind） → 横向偏移

风速	表现
微风	无偏移
中风	轻微左右摆动
大风	明显横向漂移
暴风	随机偏移 + 抖动


⸻

3.4 气压 & 稳定度 → 堆叠形态
	•	高气压 / 稳定天气：
	•	方块规则
	•	易消行
	•	低气压 / 不稳定：
	•	不规则组合
	•	难以消行

⸻

3.5 天气趋势（未来） → 关卡预览
	•	每一天 = 一个小关卡
	•	横向滑动查看
	•	背景颜色表达整体趋势

⸻

四、App 信息架构

4.1 主页面（唯一核心页）

┌───────────────┐
│ 城市 + 当前时间 │
├───────────────┤
│   Tetris 天气场 │
│   动态堆叠区   │
│               │
├───────────────┤
│ 今日一句话状态 │
│ "Today is messy"│
└───────────────┘

默认不显示数字，点击后才展开

⸻

4.2 轻量数据层（下滑展开）
	•	体感温度
	•	降水概率
	•	风速
	•	日出日落

全部为「辅助信息」。

⸻

五、Widget / 组件设计（产品灵魂）

5.1 Widget 核心原则
	•	无交互
	•	自动随天气变化
	•	一眼看情绪

⸻

5.2 Widget 类型

① 小号（4x4）
	•	单个堆叠状态
	•	代表“此刻天气”

② 中号（4x8）
	•	当前 + 接下来 6 小时
	•	方块逐列变化

③ 大号（8x8）
	•	今日完整趋势
	•	可看到消行 / 堆满

⸻

5.3 Widget 行为逻辑
	•	每小时触发一次“掉落”
	•	天气好 → 自动消一行
	•	天气差 → 堆叠混乱

⸻

六、UI 视觉设计方向（重点）

6.1 整体风格

现代极简 + 复古像素感
	•	非真实像素，而是“像素感比例”
	•	扁平 + 低饱和

⸻

6.2 配色方案（初版）

默认主题：Cold Neon
	•	背景：深灰 / 黑蓝
	•	方块：低饱和冷色
	•	强调色：青色

备选主题：Soviet Green
	•	黑底 + 荧光绿
	•	复古终端风

⸻

6.3 字体建议
	•	标题：几何无衬线（类似 DIN / Inter）
	•	数字：等宽数字
	•	文案极少

⸻

6.4 动效原则
	•	所有动画 ≤ 300ms
	•	无弹性
	•	重力感明确
	•	下落比横移重要

⸻

七、情绪文案系统（可选）

状态	文案示例
完美消行	“Clean day”
稳定	“Calm weather”
混乱	“Messy outside”
极端	“Stay inside”


⸻

八、MVP 功能范围（2–3 周）
	•	单城市天气
	•	当前 + 24 小时
	•	一个主题
	•	1–2 个 Widget
	•	无账号

⸻

九、后续可扩展方向
	•	多城市切换（多棋盘）
	•	成就系统（连续晴天）
	•	季节主题
	•	雪暴 / 极寒特效

⸻

十、设计原则总结

**这是一个“让天气变成画面”的产品，而不是一个数据工具。
